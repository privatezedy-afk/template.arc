Security Architecture

Built for the most
demanding institutions.

Every layer of ArcLight Vault is designed with the assumption that your documents are irreplaceable. Because they are.

AES-256-GCM Encryption
Every file is independently encrypted using military-grade AES-256-GCM cipher with unique per-document keys. Your data is unreadable without explicit authorization — even to us.
Encryption at Rest & Transit
Signed URL Access Control
Document downloads require cryptographically signed, time-limited URLs that expire in 60 seconds. No persistent links, no shared access — every download is a unique event.
Zero Persistent Links
Immutable Audit Log
Every vault interaction is written to a tamper-proof, cryptographically chained log. Full forensic traceability, always. WORM-compliant storage with no deletion possible.
WORM Compliant
Multi-Factor Authentication
Access requires your institutional email, a rotating secure access token, and optionally biometric or hardware key confirmation. Session isolation ensures no lateral movement.
MFA + Hardware Key
Zero-Knowledge Architecture
Encryption keys are derived from client-controlled secrets. ArcLight infrastructure never holds unencrypted document data — by design. True zero-knowledge at the application layer.
Client-Controlled Keys
Automated Integrity Scans
SHA-3 hash verification runs against all stored documents every 6 hours. Any file modification — however subtle — is detected instantly and escalated to your security contact.
6-Hour Verification Cycles
